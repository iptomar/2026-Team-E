# Fix Header Width Mismatch

## Issue
The `AppHeader` component has an outer wrapper div with `border-b` that spans the full viewport width (`w-full`). Inside it, the actual header content is constrained with `mx-auto md:max-w-7xl`. This causes the border to extend beyond the builder content (which is also constrained to `max-w-7xl` via `AppContent`), resulting in visual misalignment and black/dark bars on the sides.

## Root Cause
In `resources/js/components/app-header.tsx`:
```tsx
<div className="border-b border-sidebar-border/80">
  <div className="mx-auto flex h-16 items-center px-4 md:max-w-7xl">
    {/* header content */}
  </div>
</div>
```
The outer div is full-width while the builder and inner header are centered with max-width, creating the mismatch.

## Solution
Remove the outer wrapper and apply the border directly to the inner container, ensuring the border width matches the content width.

### Changes
**File:** `resources/js/components/app-header.tsx`

Replace:
```tsx
<div className="border-b border-sidebar-border/80">
  <div className="mx-auto flex h-16 items-center px-4 md:max-w-7xl">
    {/* ... */}
  </div>
</div>
```

With:
```tsx
<div className="mx-auto flex h-16 items-center px-4 md:max-w-7xl border-b border-sidebar-border/80">
  {/* ... */}
</div>
```

## Impact
- Header bottom border will align exactly with the constrained content width (`max-w-7xl`) across all pages.
- Eliminates the visual overflow/dark bars on the sides.
- All pages using `AppHeader` (dashboard, builder, etc.) will have consistent alignment between header and page content.
- No functional changes; purely visual fix.

## Validation
- Visual check on builder page: header border should line up with the edges of the builder content area.
- Check other pages (templates/dashboard) to confirm alignment remains correct.
- Test responsive behavior: `md:max-w-7xl` means full-width on small screens, which matches the previous behavior of the outer div (also full-width). No regression.
